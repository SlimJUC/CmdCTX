# Context Files

Context files are the local data sources that help `cmdctx` generate accurate commands for your specific machine and project. They are generated by `cmdctx init` and refreshed by `cmdctx refresh`.

---

## Machine context

Location: `~/.cmdctx/`

| File | Purpose |
|------|---------|
| `machine-context.json` | Structured machine summary (authoritative) |
| `machine-context.md` | Human-readable machine summary |

### What it contains

```json
{
  "generated_at": "2026-04-18T00:00:00Z",
  "hostname": "dev-machine",
  "os": "linux",
  "arch": "amd64",
  "home_dir": "~",
  "tools_available": {
    "grep": "/usr/bin/grep",
    "find": "/usr/bin/find",
    "jq": "/usr/bin/jq",
    "git": "/usr/bin/git"
  },
  "log_dirs": ["/var/log", "/var/log/nginx"],
  "config_dirs": ["/etc", "/etc/nginx"],
  "search_roots": ["~/projects", "~/code", "/var/www"],
  "detected_stacks": [
    {"name": "Go", "category": "language", "confidence": "high"}
  ],
  "default_excludes": [".git", "node_modules", "vendor"],
  "scan_mode": "safe",
  "scan_stats": {"total_files": 50001, "total_dirs": 15731}
}
```

### How it's used

- **Tool selection**: If `rg` is in `tools_available`, text search uses `rg`. Otherwise falls back to `grep`.
- **Log paths**: When you request a log search, log directories from context are used as targets.
- **Stack detection**: Framework signals help the AI understand your environment.
- **Search roots**: Suggested starting points for machine-wide searches.

## Project context

Location: `<project>/.cmdctx/`

| File | Purpose |
|------|---------|
| `project-context.json` | Structured project summary (authoritative) |
| `project-context.md` | Human-readable project summary |

### What it contains

```json
{
  "generated_at": "2026-04-18T00:00:00Z",
  "project_root": "~/go/myproject",
  "frameworks": [
    {"name": "Go", "category": "language", "confidence": "high"},
    {"name": "Docker", "category": "infra", "confidence": "medium"}
  ],
  "log_dirs": ["~/go/myproject/logs"],
  "config_dirs": ["~/go/myproject/config"],
  "important_files": ["~/go/myproject/go.mod", "~/go/myproject/docker-compose.yml"],
  "extensions": {".go": 42, ".yml": 5, ".md": 8},
  "ignored_dirs": ["~/go/myproject/vendor", "~/go/myproject/.git"],
  "default_excludes": [".git", "node_modules", "vendor"],
  "scan_mode": "safe",
  "notes": [
    "Detected Go (language, confidence: high)",
    "Top file types: .go(42), .md(8), .yml(5)"
  ]
}
```

## Scan modes

### Safe mode (default)

Collects only:
- File and directory names
- Extension statistics
- Important file detection (by filename)
- Directory type detection (logs, config, etc.)

No file contents are read.

### Deep mode

Everything from safe mode, plus:
- Reads content of a curated allowlist of safe manifest files:
  - `README.md`, `go.mod`, `package.json`, `composer.json`
  - `docker-compose.yml`, `Makefile`, `Dockerfile`
  - `.env.example` (never `.env`)
  - `requirements.txt`, `Cargo.toml`

**Deep mode never reads:**
- `.env` (any variant)
- Private keys (`.pem`, `.key`, `.p12`, `.pfx`)
- SSH keys (`id_rsa`, `id_ed25519`, etc.)
- Credential files
- Any file matching sensitive path heuristics

Manifest content is redacted before being stored or sent to AI.

Enable deep mode:
```bash
cmdctx init --mode deep
cmdctx refresh --mode deep
```

---

## Regenerating context

```bash
cmdctx refresh              # safe mode
cmdctx refresh --mode deep  # deep mode
cmdctx init --mode deep --project /path/to/project
```

Context is automatically re-indexed in the SQLite retrieval store after regeneration.

## Other config files

| File | Purpose |
|------|---------|
| `~/.cmdctx/config.yaml` | Main configuration |
| `~/.cmdctx/command-policy.json` | Safety policy (merged with hardcoded defaults) |
| `~/.cmdctx/history.db` | SQLite history + context chunks |

### command-policy.json

```json
{
  "prefer_tools": {
    "text_search": "rg",
    "file_search": "fd",
    "json_search": "jq"
  },
  "blocked_commands": ["rm", "sudo", ...],
  "blocked_patterns": [">", "| sh", ...],
  "default_excludes": [".git", "node_modules", "vendor"]
}
```

User additions to `blocked_commands` and `blocked_patterns` **extend** the hardcoded defaults — they cannot shrink them.
